So first off, there's two MACE evaluators that work in LAMMPS. One that goes through the unified MLIAP interface, and another which is a Kokkos implementation in a standalone package called symmetrix . Both packages perform about the same for medium size foundation models and >1000 atoms. For <1000 atoms or for small foundation models, symmetrix performs significantly better, as nearly all of the operations have been ported to GPU. I would therefore suggest using symmetrix if you're using variants of MACE-OFF, MACE-MP, MACE-MPA, MACE-OMAT or any of the other foundation models which do not have the new non-linear blocks. For the new foundation models which contain non-linear blocks (e.g, mh1), you can only use the MLIAP interface.

NOTE: NONE of the new electrostatics models have been ported to LAMMPS yet, so you can only run these in python directly. I've never done this, but I know people who have so if anyone needs them I can reach out and find out.

Do this in an interactive session on a compute node! https://docs.isambard.ac.uk/user-documentation/guides/slurm/#running-an-interactive-shell-in-an-existing-job

symmetrix : setup instructions:

1. Git clone LAMMPS and symmetrix 
git clone -b release https://github.com/lammps/lammps
git clone --recursive https://github.com/wcwitt/symmetrix

2. Load the following module env
module load PrgEnv-gnu
module load gcc-native/13.2
module load cudatoolkit/24.11_12.6
module load cray-python
module load craype-accel-nvidia90
module load cray-hdf5/1.14.3.5
module load cray-netcdf/4.9.0.17

3.  Create a python venv
python -m venv LammpsSymmetrixVenv
source LammpsSymmetrixVenv/bin/activate

4. run the ./install.sh <path_to_lammps> script in symmetrix
cd symmetrix/pair_symmetrix
./install.sh ../../lammps
cd ../..

5. Export the following environment variables
export CRAY_ACCEL_TARGET=nvidia90
export CC=$(which cc)
export CXX=$(which CC)
export FC=$(which ftn)
export NVCC_WRAPPER_DEFAULT_COMPILER=$(which CC)

6. Run the patch script in this directory (this patches the kokkos_kernels external dependency inside symmetrix, making sure that it finds the cuda libraries in Isambard.
chmod u+x /path/to/patch_symmetrix_cuda_tpls.sh 
./path/to/patch_symmetrix_cuda_tpls.sh /path/to/symmetrix


6. Build LAMMPS
cd lammps
cmake \
    -B build \
    -D CMAKE_BUILD_TYPE=Release \
    -D CMAKE_CXX_COMPILER=$(pwd)/lib/kokkos/bin/nvcc_wrapper \
    -D CMAKE_CXX_STANDARD=20 \
    -D CMAKE_CXX_STANDARD_REQUIRED=ON \
    -D CMAKE_CXX_FLAGS="-ffast-math" \
    -D BUILD_MPI=ON \
    -D BUILD_SHARED_LIBS=ON \
    -D PKG_KOKKOS=ON \
    -D Kokkos_ENABLE_SERIAL=ON \
    -D Kokkos_ENABLE_CUDA=ON \
    -D BUILD_OMP=ON \
    -D Kokkos_ARCH_HOPPER90=ON \
    -D Kokkos_ENABLE_AGGRESSIVE_VECTORIZATION=ON \
    -D SYMMETRIX_KOKKOS=ON \
    -D SYMMETRIX_SPHERICART_CUDA=ON \
    -D PKG_PYTHON=ON \
    cmake
cmake --build build -j 48
cd build
make install-python
cd ../..

Once built, this version of LAMMPS should work on multiple GPUs with symmetrix . Instructions for how to convert a mace model to be compatible with symmetrix as well as LAMMPS input script commands can be found on the github: https://github.com/wcwitt/symmetrix. Please ask me if you have questions!
